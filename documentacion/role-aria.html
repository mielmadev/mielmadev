¿Qué es ARIA?

ARIA (Accessible Rich Internet Applications) es un conjunto de atributos que mejoran la accesibilidad web, permitiendo que tecnologías de asistencia (como lectores de pantalla) comprendan mejor la función y estructura de los elementos HTML.

¿Por qué usar roles ARIA?

  • Mejora la accesibilidad para personas con discapacidad.
  • Facilita la navegación y comprensión de la estructura de la web por tecnologías de asistencia.
  • Refuerza la semántica y compatibilidad en navegadores y lectores de pantalla antiguos.

Roles ARIA más utilizados

Etiqueta HTML Ejemplo Rol ARIA ¿Implícito? ¿Cuándo usar role?
<header> <header role="banner">...</header> banner ✔️ No es necesario, salvo casos especiales (varios headers).
<nav> <nav role="navigation">...</nav> navigation ✔️ No es necesario, salvo casos especiales.
<main> <main role="main">...</main> main ✔️ No es necesario, salvo compatibilidad o varios main.
<footer> <footer role="contentinfo">...</footer> contentinfo ✔️ No es necesario, salvo varios footers.
<section> <section role="region">...</section> region ✔️ No es necesario, salvo que la sección no tenga título.
<article> <article role="article">...</article> article ✔️ No es necesario.
<aside> <aside role="complementary">...</aside> complementary ✔️ No es necesario.
<form> <form role="form">...</form> form ✔️ No es necesario.
<div> <div role="dialog">...</div> dialog ❌ Sí, siempre que sea un diálogo/modal.
<form> <form role="search">...</form> search ❌ Sí, para zonas de búsqueda.

Nota: Los roles marcados como implícitos ya están presentes en la etiqueta HTML y no es necesario añadir el atributo role, salvo casos especiales. Los roles no implícitos deben añadirse manualmente.

¿Cómo usar los roles ARIA?

  • Puedes añadir el atributo role a las etiquetas semánticas de HTML5 para reforzar la accesibilidad.
  • No hay problema en usar roles ARIA junto con etiquetas como <header>, <nav>, <main>, <footer>, <section>, <article>.
  • Ver ejemplos en la lista de roles ARIA más utilizados arriba.

Roles ARIA avanzados

Rol ARIA Descripción Ejemplo ¿Cuándo usarlo?
alert Mensaje urgente que requiere atención inmediata (errores, advertencias). <div role="alert">¡Error!</div> Para mostrar mensajes importantes que deben ser leídos por tecnologías de asistencia.
tooltip Información contextual que aparece al pasar el cursor o enfocar un elemento. <div role="tooltip">Ayuda contextual</div> Para mostrar ayudas o explicaciones breves sobre elementos interactivos.
tablist Contenedor de pestañas (tabs) en interfaces. <div role="tablist">...</div> Cuando tienes varias pestañas agrupadas.
tab Pestaña individual dentro de un tablist. <div role="tab">Pestaña 1</div> Para cada pestaña dentro de un tablist.
tree Estructura jerárquica, como un árbol de directorios. <ul role="tree">...</ul> Para listas o menús jerárquicos.
grid Tabla interactiva, similar a una hoja de cálculo. <table role="grid">...</table> Para tablas con interacción avanzada (ordenar, editar).
menu Menú de opciones, normalmente en aplicaciones o widgets. <ul role="menu">...</ul> Para menús personalizados fuera de la navegación principal.

Nota: Los roles avanzados se usan para interfaces ricas e interactivas, y ayudan a tecnologías de asistencia a interpretar correctamente la función de cada elemento.
